## 2. 🫧 评价

`github-rank` 统计了 Github 全球/中国用户排名，以及全球 Star 数最多的仓库排名，内容比 `china-ranking` 更全面。

> 🕷️Github China/Global User Ranking, Global Warehouse Star Ranking (Github Action is automatically updated daily).
>
> 🕷️Github 中国/全球 用户排名，全球仓库星标排名（Github Action 每日自动更新）。

### 3.1. 自动化触发

整个流程通过 GitHub Actions 每月 15 日 3:00 UTC 自动触发。

### 3.2. 数据采集执行顺序

按照 package.json 定义的脚本顺序执行：

- `get:trending` - 爬取 GitHub 趋势页面获取日/周/月趋势数据
- `get:repos` - 获取 stars>8000 的仓库数据
- `get:users` - 获取全球 followers>1000 的用户数据
- `get:users:china` - 获取中国地区用户数据（添加 location:China 过滤）

每个数据采集脚本都实现了 API 限流机制，通过 `sleep(1000)` 在请求间添加 1 秒间隔，避免触发 GitHub API 限制，同时也包括了错误处理和重试机制。

### 3.3. 数据处理与丰富化

`get:users:info`脚本负责数据丰富化处理：

- 合并新旧数据
- 为每个用户获取详细信息
- 数据去重和排序
- 分离用户账户和组织账户

### 3.4. 数据存储

数据先存储在`.cache/`目录作为临时缓存，最终处理后的数据保存到`dist/`目录。

### 3.5. 输出分发

最终数据通过两种方式分发：

- 生成静态HTML网页部署到GitHub Pages
- 发布为npm包`@wcj/github-rank`供程序化使用
