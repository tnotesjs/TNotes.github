## 2. 评价

tldraw 和 excalidraw 非常类似。

首先接触到的是 excalidraw，使用过程中没发现啥明显痛点，并且也习惯了 excalidraw 的交互，tldraw 就尚未去深入体验了。

## 3. 请详细解释一下 tldraw 这个项目【AI】

`tldraw/tldraw` 是一个开源的 **无限画布白板项目**，同时也是一个可嵌入到其他产品里的 **React/TypeScript 画布编辑器 SDK**。你可以把它理解成“开源版的 FigJam / Excalidraw 类工具”，但它更强调可扩展的编辑器内核。

### 3.1. 项目定位

- **在线白板应用**：官方站点是 `tldraw.com`，支持绘图、便签、文本、箭头、图片、多人协作等。
- **开发者 SDK**：通过 `@tldraw/tldraw` 等包，开发者可以把完整画布编辑器嵌入自己的 React 应用。
- **无限画布引擎**：提供缩放、平移、选择、吸附、撤销重做、形状管理、事件处理等核心能力。
- **实时协作基础**：项目包含同步、存储、schema、状态管理等底层模块，方便构建多人协作产品。

### 3.2. 核心功能

- **无限画布**：用户可以在一个无限扩展的空间里绘制、拖拽、缩放和组织内容。
- **图形编辑**：支持矩形、圆形、箭头、线条、自由绘制、文本、图片、便签等形状。
- **编辑器交互**：支持选择、框选、复制粘贴、对齐、旋转、缩放、成组、锁定等常见设计工具操作。
- **撤销重做**：内置历史记录机制。
- **文件/数据模型**：画布内容以结构化 JSON 形式保存，便于序列化、同步和迁移。
- **扩展能力**：可以自定义 shape、tool、UI、快捷键、菜单、行为逻辑等。
- **多人协作**：官方产品中支持实时多人编辑，仓库中也包含相关同步模块。

### 3.3. 技术栈

- **TypeScript**：整个项目主要用 TypeScript 编写。
- **React**：编辑器 UI 和嵌入式组件基于 React。
- **SVG / HTML / Canvas 混合渲染**：不同对象会根据需要使用不同渲染方式。
- **Monorepo**：仓库不是单个应用，而是多个 app 和 package 组成的工程。
- **npm/pnpm workspace**：通常使用 workspace 管理多个包。
- **状态管理自研**：tldraw 有自己的 reactive state/store 相关实现，用来支撑编辑器的高频交互。

### 3.4. 大致架构

- `@tldraw/tldraw`：面向开发者的主包，通常直接使用这个包嵌入完整编辑器。
- `@tldraw/editor`：编辑器核心，包含画布、工具、事件、选择、相机、形状系统等基础能力。
- `@tldraw/tlschema`：定义文档数据结构，比如 page、shape、asset、binding 等 schema。
- `@tldraw/store`：底层数据存储和记录管理。
- `@tldraw/state`：响应式状态系统。
- `@tldraw/sync` 或相关包：多人协作、客户端/服务端同步相关能力。
- `apps`：官方站点、示例、开发环境或其他应用入口。
- `packages`：可复用的核心库、工具库、编辑器模块和 UI 模块。

### 3.5. 使用方式示例

```tsx
import { Tldraw } from 'tldraw'
import 'tldraw/tldraw.css'

export default function App() {
  return <Tldraw />
}
```

这会直接在 React 应用中渲染一个完整的 tldraw 编辑器。

### 3.6. 适合用来做什么

- 在线白板
- 流程图工具
- 协作式设计草图工具
- 教学/批注工具
- 文档中的可编辑画布
- AI 工作流画布
- 可视化编辑器
- 产品原型/头脑风暴工具

### 3.7. 和 Excalidraw 的区别

- Excalidraw 更偏“手绘风白板应用”。
- tldraw 更偏“可嵌入、可扩展的编辑器框架”。
- tldraw 的 shape/tool/store/schema 抽象更系统，适合做二次开发。
- 如果你想做自己的画布产品，tldraw 通常比单纯集成一个白板应用更灵活。

### 3.8. 项目难点

- 交互复杂：选择、缩放、拖拽、旋转、吸附、快捷键等都要处理大量边界情况。
- 数据模型复杂：shape、asset、binding、page、camera、instance 状态需要分层管理。
- 性能要求高：画布编辑是高频交互，状态更新和渲染需要非常精细。
- 协作复杂：多人编辑要处理同步、冲突、离线、权限、状态广播等问题。
- 可扩展性要求高：既要支持官方白板，又要允许外部开发者自定义。

简单总结：`tldraw` 不只是一个白板网站，而是一个完整的 **无限画布编辑器平台**。如果你想研究 React 复杂交互、可扩展编辑器架构、多人协作、图形编辑器数据模型，这个项目非常值得深入看。

## 4. 从使用层面上看，你觉得 tldraw 和 excalidraw 之间的区别是什么？你会如何选择？【AI】

从**使用层面**看，我会把它们这样区分：

| 维度 | tldraw | Excalidraw |
| --- | --- | --- |
| 第一感觉 | 更像现代白板 / FigJam / 轻量设计工具 | 更像手绘草图 / 临时画图工具 |
| 视觉风格 | 干净、规整、偏产品化 | 手绘感强，非常有辨识度 |
| 上手成本 | 也不高，但功能感更强 | 极低，打开就画 |
| 适合内容 | 白板协作、流程整理、产品草图、结构化画布 | 架构图、流程图、会议草稿、临时说明图 |
| 编辑体验 | 更像完整画布编辑器，交互更“工具化” | 更轻、更直接，不容易打断思路 |
| 扩展/嵌入 | 更适合做成自己产品的一部分 | 也能嵌入，但更偏“画图组件/白板工具” |
| 风格输出 | 更正式、清爽 | 更亲和、草稿感强 |

我的直观判断是：

- **如果我是普通用户，只想快速画图、做技术说明、开会草稿，我会优先选 Excalidraw。**  
  它最大的优点是“不需要思考工具本身”。画框、连线、写字、导出，一气呵成。尤其是技术架构图、文档配图、临时流程图，Excalidraw 非常舒服。

- **如果我要做一个长期使用的白板空间，或者把画布能力嵌入自己的产品，我会优先选 tldraw。**  
  tldraw 更像一个“画布编辑器平台”，它的交互、状态、shape 扩展、工具系统更适合支撑复杂产品。比如 AI 工作流画布、在线教学白板、产品协作空间、可视化编辑器，这类场景 tldraw 更有优势。

- **如果团队成员很多、偏产品/设计/协作，我会更倾向 tldraw。**  
  它的整体气质更接近现代协作白板，适合承载持续演进的内容，而不只是一次性草图。

- **如果内容最终要放进技术文档、PR、issue、博客，我会更倾向 Excalidraw。**  
  它的手绘风格很适合表达“这是草图/概念图”，不会显得过度正式，也很容易被读者接受。

我的选择标准可以简化成一句话：

> **临时画图、快速表达、技术草稿：选 Excalidraw。  
> 长期白板、产品内嵌、复杂交互、自定义画布：选 tldraw。**

如果是我个人日常使用：

- 写技术文档、画架构图：我会用 **Excalidraw**。
- 做一个带画布能力的产品：我会用 **tldraw**。
